User's Manual

Introduction

Welcome to the T4BF MPX Operating System. This manual provides comprehensive documentation for all available commands and system functionality.

MPX (Multiprogramming Executive) is a simple operating system designed for educational purposes. This documentation will guide you through all available commands and their usage.

Getting Started

After booting MPX, you will be presented with a command prompt. From here, you can enter any of the commands documented below.

Basic Usage

Type a command and press Enter to execute it. Commands are case-insensitive.

MPX> help
MPX> version
MPX> shutdown

Command Reference

help

Syntax: help [command]

Description: Displays help information for available commands. When called without arguments, lists all commands. When called with a command name, displays detailed help for that specific command.

Parameters:
Parameter Required Description
command No The name of the command to get help for
Examples: 
# List all available commands
MPX> help

# Get help for a specific command
MPX> help shutdown

version

Syntax: version

Description: Displays the current version of MPX and compilation information.

Examples: 
MPX> version
MPX Version 1.0 - T4BF

shutdown

Syntax: shutdown

Description: Safely shuts down the MPX system. Prompts for confirmation before proceeding.

Examples: 
MPX> shutdown
Are you sure you want to shutdown? (y/n): y
Shutting down...

each command should include:

  • Command name
  • Syntax
  • Description
  • Parameters (if any)
  • Examples

Overview

  • ctype.h - Character type functions

Data Types

struct pcb

Purpose: Process Control Block - Contains all information about a process needed by the operating system.

Definition: 
struct pcb {
    char name[16];          // Process name (unique identifier)
    int class;              // Process class (system or user)
    int priority;           // Scheduling priority (0-9)
    int state;              // Current state (ready, running, blocked)
    void *stack_base;       // Base address of process stack
    void *stack_top;        // Current top of stack
    struct pcb *next;       // Pointer to next PCB in queue
    struct pcb *prev;       // Pointer to previous PCB in queue
};
Members:
Member Type Description
name char[16] Unique identifier for the process (max 15 characters + null terminator)
class int Process class: 0 = system process, 1 = user process
priority int Scheduling priority: 0 (highest) to 9 (lowest)
state int Current process state
stack_base void* Base address of allocated stack memory
stack_top void* Current stack pointer position
next struct pcb* Next PCB in the queue (linked list)
prev struct pcb* Previous PCB in the queue (linked list)

enum process_state

Purpose: Defines the possible states a process can be in.

Definition: 
enum process_state {
    READY,      // Process is ready to execute
    RUNNING,    // Process is currently executing
    BLOCKED,    // Process is waiting for an event
    SUSPENDED   // Process has been suspended
};
Values:
Value Description
READY Process is in the ready queue, waiting to be scheduled
RUNNING Process is currently being executed by the CPU
BLOCKED Process is waiting for I/O or another event
SUSPENDED Process has been manually suspended by user or system

Function Reference

pcb_alloc

Prototype:

struct pcb *pcb_alloc(void);

Description: Allocates memory for a new Process Control Block and initializes all fields to default values.

Parameters:

None

Return Value:
Value Description
struct pcb* Pointer to newly allocated PCB on success
NULL Memory allocation failed
Example: 
struct pcb *new_process = pcb_alloc();
if (new_process == NULL) {
    // Handle allocation failure
    return -1;
}
// Use the new PCB...

pcb_free

Prototype:

int pcb_free(struct pcb *pcb);

Description: Frees the memory allocated for a PCB and its associated stack. The PCB must not be in any queue when freed.

Parameters:
Parameter Type Description
pcb struct pcb* Pointer to the PCB to free. Must not be NULL.
Return Value:
Value Description
0 Success
-1 Invalid parameter (NULL pointer)
Example: 
struct pcb *process = find_pcb("myprocess");
if (process != NULL) {
    pcb_remove(process);  // Remove from queue first
    pcb_free(process);    // Then free memory
}
