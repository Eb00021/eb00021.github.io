User's Manual

Introduction

Welcome to the T4BF MPX Operating System. This manual provides comprehensive documentation for all available commands and system functionality.

MPX (Multiprogramming Executive) is a simple operating system designed for educational purposes. This documentation will guide you through all available commands and their usage.

Getting Started

After booting MPX, you will be presented with a command prompt. From here, you can enter any of the commands documented below.

Basic Usage

Type a command and press Enter to execute it. Commands are case-insensitive.

MPX> help
MPX> version
MPX> shutdown

Command Reference

help

Syntax: help [command]

Description: Displays help information for available commands. When called without arguments, lists all commands. When called with a command name, displays detailed help for that specific command.

Parameters:
Parameter Required Description
command No The name of the command to get help for
Examples: 
# List all available commands
MPX> help

# Get help for a specific command
MPX> help shutdown

version

Syntax: version

Description: Displays the current version of MPX and compilation information.

Examples: 
MP> version
MPX Version 1.0 - T4BF

shutdown

Syntax: shutdown

Description: Safely shuts down the MPX system. Prompts for confirmation before proceeding.

Examples: 
MPX> shutdown
Are you sure you want to shutdown? (y/n): y
Shutting down...

getdate

Syntax: getdate

Description: Reads the current date from the Real-Time Clock (RTC) and displays it in MM/DD/YYYY format.

Examples: 
mpx> getdate
Current date: 01/26/2026

setdate

Syntax: setdate MM/DD/YYYY

Description: Sets the system date in the RTC. The date must be in MM/DD/YYYY format.

Parameters:
Parameter Required Description
MM/DD/YYYY Yes Date to set. Month (1-12), Day (1-31), Year (1900-2099)
Examples: 
mpx> setdate 01/26/2026
Date set successfully.

gettime

Syntax: gettime

Description: Reads the current time from the RTC and displays it in HH:MM:SS format (24-hour).

Examples: 
mpx> gettime
Current time: 14:30:45

settime

Syntax: settime HH:MM:SS

Description: Sets the system time in the RTC. Time must be in HH:MM:SS format (24-hour).

Parameters:
Parameter Required Description
HH:MM:SS Yes Time to set. Hours (0-23), Minutes (0-59), Seconds (0-59)
Examples: 
mpx> settime 14:30:00
Time set successfully.

Quick Reference

Command Description Syntax
help Display help information help [command]
version Display version information version
shutdown Shutdown the system shutdown
getdate Display current date getdate
setdate Set the system date setdate MM/DD/YYYY
gettime Display current time gettime
settime Set the system time settime HH:MM:SS

Programmer's Manual

Overview

This section documents all functions, data types, and structures available in the T4BF MPX system. Use this reference to extend and modify the system.

Module Organization

  • memory.h - Memory management functions
  • processes.h - Process management and PCB operations
  • string.h - String manipulation utilities
  • stdlib.h - Standard library functions
  • ctype.h - Character type functions
  • sys_req.h - System request interface

Data Types

op_code

Purpose: Enum for system request operation codes.

Definition: 
typedef enum {
    EXIT,   // Terminate the calling process
    IDLE,   // Suspend the calling process
    READ,   // Read from a device
    WRITE,  // Write to a device
} op_code;
Values:
Value Description
EXIT Terminate the calling process (R3+)
IDLE Suspend the calling process (R3+)
READ Read data from a serial port
WRITE Write data to a serial port

device

Purpose: Device constants for serial port communication.

Definition: 
// Defined in mpx/device.h
#define COM1 0x3F8
#define COM2 0x2F8
#define COM3 0x3E8
#define COM4 0x2E8

Function Reference

serial_init

Prototype:

int serial_init(device dev);

Description: Initializes a serial port for communication. Sets baud rate to 9600, 8 data bits, no parity, 1 stop bit.

Parameters:
Parameter Type Description
dev device Serial port to initialize (COM1, COM2, etc.)
Return Value:
Value Description
0 Success
-1 Invalid device

serial_out

Prototype:

int serial_out(device dev, const char *buffer, size_t len);

Description: Writes data to a serial port.

Parameters:
Parameter Type Description
dev device Serial port to write to
buffer const char* Data to write
len size_t Number of bytes to write
Return Value:
Value Description
len Number of bytes written on success
-1 Invalid device or not initialized

serial_poll

Prototype:

int serial_poll(device dev, char *buffer, size_t len);

Description: Polls the serial port for input. Reads characters one at a time until Enter is pressed or buffer is full. Echoes input back and handles backspace.

Parameters:
Parameter Type Description
dev device Serial port to read from
buffer char* Buffer to store input
len size_t Size of the buffer
Return Value:
Value Description
>= 0 Number of bytes read
-1 Invalid buffer or length

sys_req

Prototype:

int sys_req(op_code op, ...);

Description: System request function. The main interface between user space and kernel. For READ/WRITE, additional parameters are required.

Parameters (READ/WRITE):
Parameter Type Description
op op_code Operation: READ or WRITE
dev device Device to read from or write to
buf char* Buffer for data
len size_t Buffer size / bytes to write
Return Value:
Value Description
>= 0 Number of bytes read/written
-1 Invalid operation
-2 Invalid buffer
-3 Invalid count
Example: 
char buf[128];
// Read user input
int n = sys_req(READ, COM1, buf, sizeof(buf));

// Write output
sys_req(WRITE, COM1, "Hello\r\n", 7);

sys_alloc_mem

Prototype:

void *sys_alloc_mem(size_t size);

Description: Allocates dynamic memory. This is the MPX equivalent of malloc().

Parameters:
Parameter Type Description
size size_t Number of bytes to allocate
Return Value:
Value Description
void* Pointer to allocated memory
NULL Allocation failed

sys_free_mem

Prototype:

int sys_free_mem(void *ptr);

Description: Frees memory allocated by sys_alloc_mem().

Parameters:
Parameter Type Description
ptr void* Pointer to memory to free
Return Value:
Value Description
0 Success
non-zero Error

Modules

mpx/serial.h

Serial port communication functions.

  • serial_init() - Initialize a serial port
  • serial_out() - Write to serial port
  • serial_poll() - Poll for serial input

sys_req.h

User-to-kernel interface.

  • sys_req() - System request (READ/WRITE)

memory.h

Dynamic memory allocation.

  • sys_alloc_mem() - Allocate memory
  • sys_free_mem() - Free memory

string.h

String functions (limited standard library).

  • strlen() - Get string length
  • strcmp() - Compare strings
  • memcpy() - Copy memory
  • memset() - Fill memory
  • strtok() - Tokenize string

stdlib.h

Standard library functions.

  • atoi() - String to integer

mpx/io.h

Low-level I/O macros.

  • inb() - Read byte from I/O port
  • outb() - Write byte to I/O port
